Meeting 机制

阻塞不中断 - 遇到问题时创建记录，跳过任务继续执行，最后统一处理。

工作流程

TASK-001 ✅ → TASK-002 阻塞 → 创建Meeting → 跳过 ↷
                                          ↓
TASK-003 ✅ → TASK-004 ✅ → 所有任务完成 → 处理 Meeting → 重新执行
