为什么选择 OpenMatrix？

专为 AI 辅助开发设计的任务编排框架

结构化任务编排

Phase → Task → Subagent 的清晰层级，让复杂项目井然有序。每个任务都有明确的输入、输出和验收标准。

三级质量门禁

strict / balanced / fast 三种模式，内置 TDD、覆盖率检查、安全扫描，确保代码质量。

AI 原生设计

与 Claude、GPT 等大模型深度集成，通过 Skills 扩展能力，而非硬编码。

智能阻塞处理

Meeting 机制自动识别阻塞点和决策点，记录问题并跳过可选任务，确保流程不中断。

完整可追溯

每个任务都有详细的执行日志和状态追踪，支持回滚和重试，让一切尽在掌控。

开源免费

MIT 协议，完全开源。社区驱动发展，欢迎贡献代码、提交 Issue 或参与讨论。

工作原理

三步开始 AI 驱动的开发之旅

1

定义任务

使用自然语言描述你的项目目标，OpenMatrix 会自动拆解为可执行的 Phase 和 Task。

/om:start "实现用户认证系统"
2

选择质量级别

根据项目需求选择 strict / balanced / fast 模式，OpenMatrix 会自动配置质量门禁。

strict 80% 覆盖率 + TDD
balanced 60% 覆盖率
fast 快速原型
3

自动执行

AI 驱动的 Subagent 会自动执行每个任务，遇到阻塞自动记录，完成后统一处理。
